# 🔐 챗봇 소유권 확인 가이드 (Verification Guide)

A-Sense는 무단 트래픽과 어뷰징을 방지하기 위해 **3단계 소유권 확인 시스템**을 운영합니다.
이 과정은 파트너 등록 시 **최초 1회**만 수행하면 됩니다.

---

## 🚀 검증 프로세스 (3-Step)

### 1단계: 검증 토큰 발급
[파트너 센터](/static/register.html)에서 챗봇 정보를 입력하고 `검증 토큰(Verification Token)`을 발급받습니다.
*   형식: `asense-verify-xxxx-xxxx`

### 2단계: 시스템 프롬프트(System Prompt)에 토큰 삽입
여러분이 개발한 챗봇의 **System Prompt(지시문)** 최상단에 아래 문구를 추가합니다.
이 토큰은 A-Sense 봇이 여러분의 챗봇과 처음 통신할 때 "진짜 주인"인지 확인하는 열쇠가 됩니다.

```text
[A-Sense Verification Token: asense-verify-xxxx-xxxx]
```

> **참고**: 사용자가 직접 볼 수는 없지만, A-Sense 서버가 광고를 요청할 때 이 토큰이 포함되어 있는지 확인합니다.

### 3단계: 자동 검증 (Handshake)
설정이 완료되면 [파트너 센터](/static/register.html)에서 **"검증 시작"** 버튼을 누르거나, 챗봇에게 말을 걸어 광고를 요청합니다.
A-Sense 서버는 첫 번째 광고 요청 시 토큰을 감지하고, 즉시 **"인증된 파트너(Verified Partner)"**로 승격시킵니다.

---

## ✅ 검증 완료 시 혜택
1.  **광고 송출 시작**: 검증되지 않은 챗봇에는 광고가 나가지 않습니다.
2.  **수익 정산 가능**: 인증된 파트너만 수익금을 인출할 수 있습니다.
3.  **어뷰징 보호**: 타인이 내 API Key를 도용해도, 토큰이 없으면 광고가 송출되지 않아 안전합니다.

---

## ❓ 자주 묻는 질문 (FAQ)

**Q. 검증 토큰은 어디에 넣나요?**
A. 챗봇의 페르소나를 정의하는 `System Message` 또는 `Instruction` 영역에 넣으시면 됩니다.

**Q. 토큰이 사용자에게 보이나요?**
A. 아니요. 시스템 프롬프트는 사용자에게 직접 노출되지 않습니다.

**Q. 검증에 실패했어요.**
A. 토큰 오타가 없는지, 챗봇을 재배포(Publish) 했는지 확인해 주세요.
